### 2026-05-07 前端视觉采用工作台式高信息密度界面

决策事项：

- 将前端视觉方向从低保真骨架推进到工作台式高保真界面，并在现有页面中完成第一轮落地。

背景：

- 项目已经具备学习、训练、观察池、估值、投资系统、复盘和设置等完整页面。
- 用户要求先用 imagegen 生成前端各页面美化设计图，再把这套视觉落到现有页面。
- 产品定位仍是长期使用的 A 股价值投资学习与流程工具，不是营销网站或荐股产品。

最终选择：

- 首页采用“今日工作台”作为第一屏。
- 全局采用暖白背景、深绿色主色、琥珀提醒色、克制边框和高密度卡片/表格布局。
- 使用统一的 `hero-panel`、`page-panel`、`metric-card`、`status-chip` 等样式基底。
- 4 张高保真设计图保存在 `docs/designs/`，作为后续视觉迭代参考。

选择理由：

- 工作台式布局更适合长期学习、研究、估值、检查和复盘。
- 高信息密度能减少工具型页面的跳转成本。
- 克制配色能避免把产品做成交易刺激或营销宣传风格。
- 统一样式基底能降低后续页面继续扩展时的维护成本。

服务的产品目标：

- 学习价值投资：是。
- 理解 A 股公司：是。
- 建立投资原则：是。
- 完成估值和风险判断：是。
- 执行投资纪律：是。
- 复盘和改进：是。

潜在偏离风险：

- 是否可能诱导短线交易：低。视觉不突出盘口、涨跌刺激或交易入口。
- 是否可能变成 AI 荐股：低。AI 入口仍限定为解释、质询、考官和研究助理。
- 是否可能替用户做最终决策：低。决策检查、估值和复盘仍要求用户填写最终判断。

防偏离措施：

- 不使用大面积红绿涨跌、买卖信号或推荐列表。
- 个股页面继续突出研究、估值、风险、资料来源和检查清单。
- 估值页只展示区间、安全边际、风险旗标和 AI 解释/反方质询入口。
- 复盘和投资系统页面继续强调原则、清单、版本和用户判断。

验收标准：

- `npm run typecheck` 通过。
- `npm run build` 通过。
- 首页、学习、AI 导师、训练、观察池、估值、投资系统、复盘、设置页面共享统一视觉语言。
- 页面仍不出现自动荐股、今日买入、买卖信号或自动交易入口。

### 2026-05-04 P1 数据源接入先做估值快照

决策事项：

- 将外部数据源接入的第一步限定为股票搜索、A 股代码标准化和东方财富估值快照，不直接迁移完整行情终端能力。

背景：

- 用户希望降低估值填写复杂度，未来可以输入公司名称或代码后自动获取财报和行情数据。
- 从 `jcp` 项目评估看，东方财富估值快照能直接提供当前价、PE TTM、PB、市值、股本等估值基础字段。

最终选择：

- P1 Issue #67 先接股票搜索和估值快照，并在估值表单中作为“行情取数助手”使用。
- TDX、Sina、K 线、盘口和完整 F10 财报放到后续 provider 扩展。

选择理由：

- 当前估值页最需要的是当前价和总股本自动填充。
- 估值快照比盘口行情更贴近价值投资主流程。
- 小步接入可以降低外部数据源不稳定对项目的影响。

服务的产品目标：

- 学习价值投资：是。
- 理解 A 股公司：是。
- 建立投资原则：间接支持。
- 完成估值和风险判断：是。
- 执行投资纪律：是。
- 复盘和改进：间接支持。

潜在偏离风险：

- 是否可能诱导短线交易：中。
- 是否可能变成 AI 荐股：中。
- 是否可能替用户做最终决策：低。

防偏离措施：

- 数据助手只填入事实字段和来源说明，不生成买卖信号。
- 保存估值前仍要求用户确认三情景假设。
- 页面继续声明数据用于估值辅助，不构成投资建议。

验收标准：

- 输入公司名称或代码可以搜索 A 股公司。
- 选中公司后可以获取当前价、PE/PB、市值和股本快照。
- 用户点击确认后才把数据写入表单。
- 保存估值仍只输出区间、假设、风险和安全边际。

### 2026-05-03 MVP 主流程完成后进入 P1 后半段

决策事项：

- 将项目状态从 MVP 开发阶段更新为 P1 专业化增强阶段，并明确下一批不再重复建设基础闭环。

背景：

- MVP 主流程已经可以完成学习、训练、模型配置、观察池、估值、检查清单、决策日志、复盘和本地备份。
- P1 已完成多项增强，包括 AI 资料提取、AI 估值草稿、导入导出、自定义检查清单、原则版本历史、决策详情、估值敏感性图表和真实 A 股案例库第一批。

最终选择：

- 后续优先做联网搜索/数据源接入、估值草稿自动化、真实 A 股案例第二批、训练题库扩展和自动化回归检查。
- 多用户账号、PostgreSQL、审计日志和部署方案进入开放化准备阶段，不进入当前本地优先迭代。

选择理由：

- 当前产品已经具备可用闭环，继续堆基础功能的边际收益下降。
- 用户反馈的痛点集中在数据录入成本、案例专业度、估值填写复杂度和长期稳定性。
- 先建立数据来源、字段映射和回归检查，可以降低后续开放化改造风险。

服务的产品目标：

- 学习价值投资：是。
- 理解 A 股公司：是。
- 建立投资原则：是。
- 完成估值和风险判断：是。
- 执行投资纪律：是。
- 复盘和改进：是。

潜在偏离风险：

- 是否可能诱导短线交易：中。联网搜索和数据源接入容易让用户期待“自动判断”。
- 是否可能变成 AI 荐股：中。真实公司案例和数据源必须保持研究训练边界。
- 是否可能替用户做最终决策：中。AI 估值草稿不能变成自动买卖结论。

防偏离措施：

- 数据源只提供事实、字段、日期和来源，不生成交易信号。
- AI 估值草稿必须由用户确认关键参数后才能保存。
- 个股案例继续用于学习研究框架、证据清单和反方问题，不输出买入、卖出或持有建议。
- 后续每个功能仍通过 Issue、分支和 PR 管理，并保留回退方案。

验收标准：

- 新增数据能力保留来源链接、数据日期和待确认状态。
- 估值工具只输出区间、假设、风险和安全边际。
- 页面不出现自动荐股、今日买入、买卖信号或自动交易入口。

### 2026-05-01 产品定位为教育、训练与流程系统

决策事项：

- 产品定位为 A 股价值投资 AI 导师，而不是 AI 荐股系统。

背景：

- 用户希望通过 AI 学习价值投资，并最终搭建一套自运行的投资系统。
- 用户希望未来开放给其他人使用，因此需要提前明确合规与产品边界。

最终选择：

- 产品围绕学习、训练、估值、观察池、投资原则、决策清单和复盘展开。
- AI 作为耐心导师和投资委员会反方委员，不直接替用户做买卖决定。

选择理由：

- 该方向更符合价值投资长期训练的本质。
- 可以降低产品跑偏成荐股工具的风险。
- 也更适合未来开放化。

服务的产品目标：

- 学习价值投资：是。
- 理解 A 股公司：是。
- 建立投资原则：是。
- 完成估值和风险判断：是。
- 执行投资纪律：是。
- 复盘和改进：是。

潜在偏离风险：

- 是否可能诱导短线交易：低。
- 是否可能变成 AI 荐股：中，需要持续约束。
- 是否可能替用户做最终决策：中，需要 AI 输出边界。

防偏离措施：

- 建立 AI 角色与边界规范。
- 所有个股分析回到用户原则、数据来源、估值假设和风险清单。
- 禁止输出直接买入或卖出指令。

验收标准：

- MVP 中 AI 不直接输出“应该买入/卖出某股票”。
- 每个投资动作都有用户最终判断记录。

### 2026-05-01 第一版聚焦中国 A 股

决策事项：

- 第一版市场范围聚焦中国 A 股。

背景：

- 用户明确希望服务中国 A 股投资。

最终选择：

- 学习内容采用经典价值投资体系加 A 股案例本土化。
- 估值工具覆盖金融、周期、消费、制造四类 A 股常见公司模板。

选择理由：

- A 股市场结构、财报披露、行业周期和政策环境与美股不同。
- 聚焦 A 股可以让案例、工具和流程更贴近用户真实使用场景。

防偏离措施：

- MVP 不做全球市场泛化。
- 非 A 股内容只作为价值投资思想源流，不作为第一版数据和工具重点。

验收标准：

- 观察池以 A 股股票代码、交易所和 A 股行业分类为核心。
- 估值模板能够覆盖金融、周期、消费、制造四类公司。

### 2026-05-01 模型由用户自定义接入

决策事项：

- 系统不内置模型服务和 API Key，由用户自行配置模型提供商。

背景：

- 用户希望通过配置模型提供商的方式接入 AI。

最终选择：

- 建立模型配置中心。
- 支持自定义 API Base URL、API Key、模型名称和角色用途。

选择理由：

- 用户对模型选择拥有控制权。
- 便于未来支持本地模型、兼容 OpenAI API 的服务和不同云模型。
- 降低产品对单一模型服务商的绑定。

防偏离措施：

- API Key 加密保存。
- 不在日志中记录 API Key。
- 不把模型能力包装成投资顾问能力。

验收标准：

- 用户可以完成模型连接测试。
- 用户可以为导师、考官、反方委员、总结等角色选择模型。
