### 2026-05-03 MVP 主流程完成后进入 P1 后半段

决策事项：

- 将项目状态从 MVP 开发阶段更新为 P1 专业化增强阶段，并明确下一批不再重复建设基础闭环。

背景：

- MVP 主流程已经可以完成学习、训练、模型配置、观察池、估值、检查清单、决策日志、复盘和本地备份。
- P1 已完成多项增强，包括 AI 资料提取、AI 估值草稿、导入导出、自定义检查清单、原则版本历史、决策详情、估值敏感性图表和真实 A 股案例库第一批。

最终选择：

- 后续优先做联网搜索/数据源接入、估值草稿自动化、真实 A 股案例第二批、训练题库扩展和自动化回归检查。
- 多用户账号、PostgreSQL、审计日志和部署方案进入开放化准备阶段，不进入当前本地优先迭代。

选择理由：

- 当前产品已经具备可用闭环，继续堆基础功能的边际收益下降。
- 用户反馈的痛点集中在数据录入成本、案例专业度、估值填写复杂度和长期稳定性。
- 先建立数据来源、字段映射和回归检查，可以降低后续开放化改造风险。

服务的产品目标：

- 学习价值投资：是。
- 理解 A 股公司：是。
- 建立投资原则：是。
- 完成估值和风险判断：是。
- 执行投资纪律：是。
- 复盘和改进：是。

潜在偏离风险：

- 是否可能诱导短线交易：中。联网搜索和数据源接入容易让用户期待“自动判断”。
- 是否可能变成 AI 荐股：中。真实公司案例和数据源必须保持研究训练边界。
- 是否可能替用户做最终决策：中。AI 估值草稿不能变成自动买卖结论。

防偏离措施：

- 数据源只提供事实、字段、日期和来源，不生成交易信号。
- AI 估值草稿必须由用户确认关键参数后才能保存。
- 个股案例继续用于学习研究框架、证据清单和反方问题，不输出买入、卖出或持有建议。
- 后续每个功能仍通过 Issue、分支和 PR 管理，并保留回退方案。

验收标准：

- 新增数据能力保留来源链接、数据日期和待确认状态。
- 估值工具只输出区间、假设、风险和安全边际。
- 页面不出现自动荐股、今日买入、买卖信号或自动交易入口。

### 2026-05-02 开发前关键选择确认

决策事项：

- 确认 MVP 技术栈、AI 接入边界、数据形态、内容策略和开发顺序。

背景：

- 用户已经确认项目从文档阶段进入路线图和项目骨架阶段。

最终选择：

- 技术栈采用 `Next.js + TypeScript + SQLite + Drizzle`。
- UI 采用 `Tailwind CSS + shadcn/ui 风格 + lucide-react`。
- 第一阶段只支持 OpenAI API 兼容格式。
- MVP 不内置联网搜索，联网搜索放 P1。
- 第一版不做登录注册。
- 本地数据支持 JSON 导出。
- A 股案例可以使用真实公司案例。
- 估值输出区间、假设、风险和安全边际，不输出买入建议。

选择理由：

- 该方案能最快完成本地可用 MVP。
- OpenAI API 兼容格式能覆盖较多模型提供商。
- 暂不做联网搜索能降低第一版复杂度和数据来源风险。
- 不做登录注册符合单用户本地优先定位。

服务的产品目标：

- 学习价值投资：是。
- 理解 A 股公司：是。
- 建立投资原则：是。
- 完成估值和风险判断：是。
- 执行投资纪律：是。
- 复盘和改进：是。

潜在偏离风险：

- 是否可能诱导短线交易：低。
- 是否可能变成 AI 荐股：中。
- 是否可能替用户做最终决策：中。

防偏离措施：

- 保持 AI 角色与边界规范。
- 页面不提供推荐股票、买卖信号或今日买入列表。
- 决策日志必须由用户填写最终判断。
- 估值结果不生成买入建议。

验收标准：

- MVP 可在本地浏览器运行。
- AI 功能只通过用户自配 OpenAI API 兼容模型调用。
- 个股相关输出不包含直接买卖指令。

### 2026-05-01 产品定位为教育、训练与流程系统

决策事项：

- 产品定位为 A 股价值投资 AI 导师，而不是 AI 荐股系统。

背景：

- 用户希望通过 AI 学习价值投资，并最终搭建一套自运行的投资系统。
- 用户希望未来开放给其他人使用，因此需要提前明确合规与产品边界。

最终选择：

- 产品围绕学习、训练、估值、观察池、投资原则、决策清单和复盘展开。
- AI 作为耐心导师和投资委员会反方委员，不直接替用户做买卖决定。

选择理由：

- 该方向更符合价值投资长期训练的本质。
- 可以降低产品跑偏成荐股工具的风险。
- 也更适合未来开放化。

服务的产品目标：

- 学习价值投资：是。
- 理解 A 股公司：是。
- 建立投资原则：是。
- 完成估值和风险判断：是。
- 执行投资纪律：是。
- 复盘和改进：是。

潜在偏离风险：

- 是否可能诱导短线交易：低。
- 是否可能变成 AI 荐股：中，需要持续约束。
- 是否可能替用户做最终决策：中，需要 AI 输出边界。

防偏离措施：

- 建立 AI 角色与边界规范。
- 所有个股分析回到用户原则、数据来源、估值假设和风险清单。
- 禁止输出直接买入或卖出指令。

验收标准：

- MVP 中 AI 不直接输出“应该买入/卖出某股票”。
- 每个投资动作都有用户最终判断记录。

### 2026-05-01 第一版聚焦中国 A 股

决策事项：

- 第一版市场范围聚焦中国 A 股。

背景：

- 用户明确希望服务中国 A 股投资。

最终选择：

- 学习内容采用经典价值投资体系加 A 股案例本土化。
- 估值工具覆盖金融、周期、消费、制造四类 A 股常见公司模板。

选择理由：

- A 股市场结构、财报披露、行业周期和政策环境与美股不同。
- 聚焦 A 股可以让案例、工具和流程更贴近用户真实使用场景。

防偏离措施：

- MVP 不做全球市场泛化。
- 非 A 股内容只作为价值投资思想源流，不作为第一版数据和工具重点。

验收标准：

- 观察池以 A 股股票代码、交易所和 A 股行业分类为核心。
- 估值模板能够覆盖金融、周期、消费、制造四类公司。

### 2026-05-01 模型由用户自定义接入

决策事项：

- 系统不内置模型服务和 API Key，由用户自行配置模型提供商。

背景：

- 用户希望通过配置模型提供商的方式接入 AI。

最终选择：

- 建立模型配置中心。
- 支持自定义 API Base URL、API Key、模型名称和角色用途。

选择理由：

- 用户对模型选择拥有控制权。
- 便于未来支持本地模型、兼容 OpenAI API 的服务和不同云模型。
- 降低产品对单一模型服务商的绑定。

防偏离措施：

- API Key 加密保存。
- 不在日志中记录 API Key。
- 不把模型能力包装成投资顾问能力。

验收标准：

- 用户可以完成模型连接测试。
- 用户可以为导师、考官、反方委员、总结等角色选择模型。
