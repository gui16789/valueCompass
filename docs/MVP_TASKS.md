# MVP 开发任务拆分

版本：v0.4

状态：MVP 主流程已完成，P1 增强进行中

## 1. 目标

MVP 的目标不是一次性做成完整投资平台，而是完成一条可用闭环：

用户学习价值投资概念，接受 AI 测验，配置自己的 AI 模型，建立 A 股观察池，完成一次估值，按投资原则做检查，记录一次用户自己的投资判断，并能复盘。

截至 2026-05-03，MVP 主流程已完成，并已完成多项 P1 增强。

## 2. MVP 验收主流程

MVP 完成后，用户应能完成：

1. [x] 打开本地浏览器网站。
2. [x] 查看价值投资时间线和学派地图。
3. [x] 打开一个知识节点并向 AI 导师提问。
4. [x] 完成一个概念测验并收到 AI 反馈。
5. [x] 配置一个模型提供商并通过连接测试。
6. [x] 手动新增一家 A 股公司到观察池。
7. [x] 为该公司添加研究摘要和资料来源。
8. [x] 选择一个估值模板并录入三情景参数。
9. [x] 请求 AI 解释估值和反方质疑。
10. [x] 建立个人投资原则。
11. [x] 运行买入、卖出或不操作检查清单。
12. [x] 保存最终用户判断为决策日志。
13. [x] 创建一次复盘记录。

## 3. 任务分组

### 3.1 项目基础

完成状态：已完成。

任务：

- 初始化前端项目。
- 初始化本地服务端。
- 初始化本地数据库。
- 建立基础路由。
- 建立页面布局。
- 建立设计系统基础组件。
- 建立本地配置读取机制。

验收标准：

- 本地开发服务器可以启动。
- 浏览器可以访问首页。
- 页面之间可以导航。
- 数据库可以创建和读取基础数据。

### 3.2 数据层

完成状态：已完成 MVP 数据层，并已补充学习进度、训练结果、资料来源、原则版本、导入导出相关数据能力。

任务：

- 实现模型配置数据表。
- 实现知识节点数据表。
- 实现学习进度和测验记录。
- 实现公司和观察池数据表。
- 实现资料来源数据表。
- 实现估值记录数据表。
- 实现投资原则数据表。
- 实现检查清单和检查运行数据表。
- 实现决策日志数据表。
- 实现复盘数据表。
- 实现 AI 会话和消息记录。

验收标准：

- 所有 P0 页面需要的数据可以保存和读取。
- 删除操作不误删关键历史记录。
- API Key 不以明文形式暴露在页面或日志中。

### 3.3 模型配置中心

完成状态：已完成，并已支持已保存提供商编辑、删除、局部连接测试和 TLS 高级开关。

任务：

- 新增模型提供商表单。
- 支持 API Base URL、API Key、模型名称。
- 支持 OpenAI API 兼容配置。
- 支持导师、考官、反方委员、研究助理角色分配。
- 实现连接测试。

验收标准：

- 用户可以配置至少一个模型提供商。
- 用户可以测试连接是否成功。
- 用户可以为 AI 角色选择不同模型。
- 未配置模型时，AI 功能给出清晰提示。

### 3.4 AI 服务层

任务：

- 封装模型调用接口。
- 支持按角色读取模型配置。
- 实现导师模式提示词。
- 实现反方委员模式提示词。
- 实现考官模式提示词。
- 实现研究助理模式提示词。
- 保存 AI 会话和消息。
- 对买卖指令类输出增加边界提示。

验收标准：

- 四种角色可以正常对话。
- AI 输出符合 `AI_ROLE_POLICY.md`。
- 个股分析不会直接输出买入或卖出结论。
- 对话可关联知识节点、公司、估值或决策。

验收标准：

## 6. 风险与应对

### 6.1 AI 幻觉

风险：

- AI 可能生成错误公司信息或错误解释。

应对：

- 保留来源。
- 标记待确认。
- 重要数据要求用户确认。
- 个股分析区分事实、推断、观点和待确认内容。

### 6.2 产品跑偏成荐股工具

风险：

- 用户可能直接问“能不能买”。

应对：

- AI 角色边界。
- 检查清单强制用户填写最终判断。
- 不提供推荐股票页面。
- 不提供买卖信号入口。

### 6.3 估值工具过复杂

风险：

- 新手无法使用专业模板。

应对：

- 每个参数提供解释。
- AI 导师随时解释。
- 提供示例数据。
- 支持先用简化模式，再展开高级参数。

### 6.4 数据维护成本高

风险：

- 手动录入过多，用户难以坚持。

应对：

- 支持粘贴资料后 AI 提取。
- 表单字段分为必填和可选。
- 先记录投资判断所需最小数据。
- 后续再考虑公开数据源接入。

- 是否使用 SQLite。

- 是否第一版支持联网搜索。
- 知识节点第一批内容范围。
- 四类估值模板的最小输入字段。
- 本地启动和数据备份方式。
