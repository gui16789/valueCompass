# 项目路线图

版本：v0.5

状态：已同步至 2026-05-04 本地开发进度

## 1. 已确认关键选择

用户已确认：

- 技术栈采用 `Next.js + TypeScript + SQLite + Drizzle`。
- UI 采用 `Tailwind CSS + shadcn/ui 风格 + lucide-react`。
- 本地启动方式接受 `npm run dev`，浏览器访问 `http://localhost:3000`。
- 本地数据库可以放在项目目录，例如 `data/local.db`。
- 第一阶段只支持 OpenAI API 兼容格式。
- MVP 不内置联网搜索，联网搜索放 P1。
- 第一批知识节点按内容计划推进。
- A 股案例可以使用真实公司案例，但必须保持教育用途和非投资建议边界。
- 估值工具采用简化模式加高级字段折叠。
- 估值结果只输出区间、假设、风险和安全边际，不输出买入建议。
- 第一版不做登录注册。
- MVP 支持本地数据备份和 JSON 导出。
- 开发顺序按本文路线图推进。

## 3. 总体路线

路线图分为 5 个里程碑：

- M0：项目骨架与基础设施。
- M1：AI 模型配置与 AI 服务层。
- M2：学习地图与训练闭环。
- M3：观察池、投资原则、检查清单与决策日志。
- M4：估值工具、复盘与本地备份。

MVP 完成标志：

用户可以完成一次完整流程：学习一个概念，接受 AI 测验，新增一家公司到观察池，完成估值，运行检查清单，保存自己的最终判断，并创建复盘记录。

当前状态：MVP 主流程已完成，正在进入 P1/P2 专业化与开放化准备。

## 4. M0：项目骨架与基础设施

目标：

- 建立可运行的 Next.js 本地项目。
- 建立目录结构、基础布局和页面路由。
- 建立数据库和类型校验基础。

任务：

- 初始化 `Next.js App Router`。
- 配置 TypeScript。
- 配置 Tailwind CSS。
- 配置 ESLint。
- 配置 SQLite 与 Drizzle。
- 建立 `data/local.db` 本地数据库位置。
- 建立基础导航和页面布局。
- 建立空页面路由。
- 建立基础 UI 组件。

验收标准：

- `npm run dev` 可以启动。
- 访问 `http://localhost:3000` 能看到首页。
- 主导航包含学习地图、AI 导师、训练场、观察池、估值工具、投资系统、复盘、设置。
- 页面路由可访问。

完成状态：已完成。

## 5. M1：模型配置与 AI 服务层

目标：

- 用户可以配置 OpenAI API 兼容模型。
- 系统可以按角色调用 AI。

任务：

- 建立模型提供商表。
- 建立模型角色配置表。
- 实现 API Key 本地加密。
- 实现模型配置页面。
- 实现连接测试。
- 实现 OpenAI API 兼容请求适配器。
- 实现导师、反方委员、考官、研究助理四类提示词。
- 实现 AI 会话和消息保存。

验收标准：

- 用户可以新增模型提供商。
- 用户可以测试模型连接。
- 用户可以为四类 AI 角色指定模型。
- 未配置模型时页面给出清晰提示。
- AI 不直接输出买入或卖出建议。

完成状态：已完成。已支持 OpenAI API 兼容提供商配置、连接测试局部更新、已保存提供商编辑、TLS 跳过高级开关和角色模型配置。

目标：

- 用户可以建立 A 股观察池和个人投资原则。

- 实现公司研究详情。
- 实现资料来源管理。
- 实现粘贴资料后 AI 辅助提取。
- 实现投资原则编辑器。
- 实现买入检查清单。
- 实现卖出检查清单。
- 实现不操作检查清单。

- 实现决策日志。

验收标准：

目标：

- 用户可以使用四类专业估值模板。
- 用户可以记录复盘并导出本地数据。

任务：

- 实现金融股估值模板。
- 实现周期股估值模板。
- 实现消费股估值模板。
- 实现制造业估值模板。
- 实现通用 DCF。
- 实现反向 DCF。
- 实现安全边际计算。
- 实现简化模式和高级字段折叠。
- 实现 AI 估值解释。
- 实现 AI 反方质疑。
- 实现复盘记录。
- 实现 JSON 导出。

验收标准：

- 四类模板都能创建估值记录。
- 每个模板支持悲观、中性、乐观三情景。
- 输出为区间、假设、风险和安全边际。
- 页面不输出买入建议。
- 用户可以为决策创建复盘。
- 用户可以导出本地数据。

完成状态：已完成。已支持四类估值模板、三情景、AI 估值草稿、敏感性图表、关键变量矩阵、本地 JSON 导入导出和复盘记录。

## 11. 防偏离红线

以下能力不进入 MVP：

- 自动荐股。
- 今日买入列表。
- 买卖信号。
- 自动交易。
- 收益排行。
- 社区跟单。

所有涉及个股的 AI 输出必须回到：

- 用户原则。
- 数据来源。
- 估值假设。
- 风险清单。
- 用户最终判断。
