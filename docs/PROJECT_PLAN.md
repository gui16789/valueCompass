# 项目计划

版本：v0.4

状态：已同步至 2026-05-03 本地开发进度

## 1. 项目目标

构建一个本地优先、浏览器访问的 A 股价值投资 AI 导师网站。第一阶段聚焦单用户本地使用，后续预留多用户开放能力。

项目不从代码开始，而是先用文档固化产品边界、阶段计划、AI 行为边界和数据结构，降低后续开发跑偏风险。

## 2. 阶段规划

当前进度摘要：

- 阶段 0 到阶段 4 的 MVP 主流程已完成。
- 阶段 3 和阶段 4 已完成多项 P1 增强，包括公司研究详情、资料来源管理、自定义检查清单、原则版本历史、AI 估值草稿、敏感性图表、导入导出和真实 A 股案例库第一批。
- 当前尚未进入阶段 5 的开放化开发，相关事项仍为方案设计和待拆分 Issue。

### 阶段 0：需求与设计冻结

目标：确认产品边界和 MVP 范围。

产出：

- PRD。
- 项目计划。
- AI 角色与合规边界。
- 数据模型。
- 页面清单。
- MVP 验收标准。

完成标准：

- 用户确认产品定位。
- 用户确认 MVP 功能范围。
- 用户确认哪些功能不进入第一版。
- 用户确认 AI 不直接荐股或替用户决策。

完成状态：已完成。

### 阶段 1：学习系统原型

目标：完成价值投资知识学习主线。

范围：

- 首页价值投资时间线。
- 学派地图。
- 人物、书籍、概念知识卡。
- A 股本土案例卡。
- 学习路径。
- AI 导师解释模式。

完成标准：

- 用户进入首页即可看到价值投资发展脉络。
- 至少 20 个知识节点可浏览。
- 用户可以围绕知识节点向 AI 提问。
- AI 可以用通俗方式解释概念，并关联 A 股案例。

完成状态：已完成，并已扩展到 61 个知识节点和真实 A 股案例库第一批；第二批已启动，新增华友钴业周期制造估值教学案例。

### 阶段 2：训练系统

目标：通过问答和案例验证用户是否理解。

范围：

- 概念测验。
- 场景判断题。
- 案例分析题。
- AI 追问。
- 掌握度评分。
- 错题和复习建议。

完成标准：

- 用户可以完成一个学习主题后的测验。
- AI 可以识别回答中的概念漏洞。
- AI 可以给出下一步学习建议。
- 系统可以记录学习进度。

完成状态：已完成。

### 阶段 3：投资流程系统

目标：搭建个人投资系统的流程骨架。

范围：

- A 股观察池。
- 投资原则编辑器。
- 买入检查清单。
- 卖出检查清单。
- 不操作检查清单。
- 决策日志。
- 复盘记录。

完成标准：

- 用户可以手动维护 A 股观察池。
- 用户可以配置自己的投资原则。
- 每次决策都可以绑定原则、估值和反方意见。
- 用户可以记录复盘结论。

完成状态：已完成，并已增强公司研究详情、资料来源管理、自定义检查清单、原则版本历史和决策详情。

### 阶段 4：估值系统

目标：完成专业估值工具第一版。

范围：

- 金融股估值模板。
- 周期股估值模板。
- 消费股估值模板。
- 制造业估值模板。
- DCF。
- 所有者收益估值。
- PE 情景估值。
- PB-ROE 估值。
- 股息折现。
- 反向 DCF。
- 安全边际计算。
- 敏感性分析。
- AI 估值解释。

完成标准：

- 用户可以为四类公司创建估值记录。
- 每个估值模板都支持悲观、中性、乐观情景。
- AI 可以解释估值参数的含义和敏感性。
- 系统不会直接输出买卖指令。

完成状态：已完成，并已增强 AI 估值草稿、敏感性图表、关键变量矩阵和本地导入导出。

### 阶段 5：开放化准备

目标：为未来开放给其他人使用做准备。

范围：

- 用户系统。
- 权限隔离。
- 用户模型配置隔离。
- 数据备份。
- 合规提示。
- 审计日志。
- 部署配置。

完成标准：

- 多用户数据互相隔离。
- 模型配置按用户隔离。
- 关键 AI 输出可追溯。
- 产品具备明确教育用途和非投资顾问边界。

完成状态：未开始。当前仅保留开放化准备项，不进入本地 MVP。

## 3. 推荐技术方向

此处是方向建议，正式开发前需要再确认。

### 3.1 前端

建议使用浏览器网站形态，便于本地访问和未来开放：

- React 或 Next.js。
- TypeScript。
- Tailwind CSS 或项目内设计系统。
- 适合时间线、地图、表格和表单的组件库。

### 3.2 后端

本地优先，后续可迁移：

- Node.js 服务端。
- SQLite 作为本地数据库。
- 后续可迁移 PostgreSQL。
- 本地加密存储模型配置。

### 3.3 AI 接入

采用模型提供商配置中心：

- 支持 OpenAI API 兼容格式。
- 支持自定义 Base URL。
- 支持按角色选择模型。
- 支持连接测试。
- 后续支持本地模型。

### 3.4 数据获取

MVP 以用户手动维护为主：

- 手动录入。
- 粘贴公告、年报、财报文本。
- AI 辅助提取。
- 联网搜索作为可选能力。
- 来源必须保留。

## 6. 项目不偏离机制

### 6.1 功能准入清单

新增功能必须回答：

- 它服务学习、训练、投资流程中的哪一部分？
- 它是否增强用户自己的判断力？
- 它是否帮助用户形成原则或复盘？
- 它是否可能诱导用户短线交易？
- 它是否会让产品变成 AI 荐股工具？

### 6.2 决策日志

项目开发中维护产品决策记录：

- 决策事项。
- 决策原因。
- 替代方案。
- 为什么当前阶段不做其他方案。
- 对 MVP 范围的影响。

### 6.3 验收驱动

每个阶段开工前必须有验收标准。没有验收标准的功能不进入开发。

### 6.4 合规提醒

涉及个股分析、买入、卖出、持仓建议时，AI 输出必须回到：

- 用户原则。
- 数据来源。
- 估值假设。
- 风险提示。
- 检查清单。
- 用户自行决策。
