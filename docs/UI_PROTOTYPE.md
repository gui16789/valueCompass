# 前端原型说明

版本：v0.4

状态：低保真原型，待评审

## 1. 当前结论

目前项目已有可运行页面骨架，但在本文件之前还没有专门的前端原型图。v0.4 增加两类原型：

- 文档低保真线框：用于确认页面信息层级。
- 浏览器静态原型页：访问 `/prototype` 查看核心工作台布局。

原型目标不是做营销型首页，而是先验证产品作为学习与投资流程工具是否清楚、克制、可长期使用。

## 2. 设计原则

- A 股投资工具应偏工作台，而不是落地页。
- 第一屏要让用户看到学习进度、AI 模型状态、观察池和待办事项。
- 学习地图和投资系统并重，避免变成单纯聊天工具。
- “不操作”与买入、卖出同等重要。
- 个股页面突出研究、估值、风险、原则匹配，不突出交易刺激。
- 所有个股相关结论都回到用户原则和检查清单。

## 3. 信息密度

页面风格建议：

- 安静、清晰、专业。
- 使用浅色背景和明确分区。
- 卡片用于单个信息块，不做层层嵌套。
- 表格、清单、分栏优先于大段说明文字。
- 关键操作按钮使用图标加文字。
- 复杂工具默认简化模式，高级字段折叠。

## 10. 待确认问题

后续视觉设计需要继续确认：

- 首页是否直接采用“工作台”作为第一屏。
- 学习地图是横向时间线优先，还是学派关系图优先。
- AI 导师是否常驻右侧抽屉，还是独立页面。
- 观察池列表是否优先做表格密度，还是看板分组。
- 估值工具是否默认双栏：输入在左，输出和 AI 解释在右。
