# 第一批知识节点内容计划

版本：v0.3

状态：草案

## 1. 目标

第一批知识节点用于支撑 MVP 的学习地图、测验和 AI 导师上下文。内容不追求百科全书式完整，而是先覆盖价值投资入门到实践系统搭建所需的关键路径。

## 2. 内容原则

- 经典体系为骨架。
- A 股案例用于本土化理解。
- 每个节点都要能连接到测验或实践动作。
- 避免把内容写成市场观点或荐股。
- A 股案例只用于说明概念，不作为投资建议。

## 3. 节点类型

节点类型包括：

- 时间线节点。
- 学派节点。
- 人物节点。
- 书籍节点。
- 概念节点。
- A 股案例节点。

## 4. MVP 最小节点数量

建议第一版至少 40 个节点：

- 时间线节点：8 个。
- 学派节点：8 个。
- 人物节点：8 个。
- 书籍节点：6 个。
- 概念节点：14 个。
- A 股案例节点：8 个。

数量超过 40 是因为一个高质量学习地图需要有足够连接点，但每个节点的正文可以先控制在中等长度。

## 10. A 股案例节点

第一批案例只作为教学案例，不作为投资建议。

建议覆盖：

- 白酒消费公司案例：说明品牌、渠道、提价能力和高 ROE。
- 银行案例：说明 PB、ROE、不良率和拨备。
- 保险案例：说明内含价值和长期负债。
- 煤炭或资源品案例：说明周期中枢利润。
- 化工案例：说明周期与产能。
- 家电案例：说明成熟消费制造、现金流和竞争格局。
- 新能源制造案例：说明高增长、竞争、资本开支和估值风险。
- 医药或创新药案例：说明研发投入、不确定性和估值边界。

案例字段：

- 公司类型。
- 教学主题。
- 适合说明的概念。
- 需要避免的误解。
- 可连接的估值模板。
- 数据来源要求。

## 11. 每个节点的内容结构

建议统一结构：

```text
标题
一句话解释
核心内容
为什么重要
常见误解
A 股本土化说明
相关节点
推荐测验
```

## 12. 内容生产顺序

推荐顺序：

1. 概念节点。
2. 时间线节点。
3. 学派节点。
4. 人物节点。
5. 书籍节点。
6. A 股案例节点。

原因：

- 概念节点最常被 AI 导师、测验和估值工具引用。
- 时间线和学派负责组织知识地图。
- 案例节点需要更谨慎地处理来源和边界。

## 13. 内容验收标准

每个节点完成后应满足：

- 能被普通新手理解。
- 有至少一个相关节点。
- 有至少一个测验入口。
- 能连接到一个实践动作或估值工具。
- 不包含直接投资建议。
- A 股案例有来源要求或待补来源标记。
